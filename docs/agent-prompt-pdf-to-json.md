# 給 AI Agent 的提示詞：把掃描 PDF 課文轉換為 Study Guy JSON

> **分兩階段使用：**
> - **階段 A（外部 AI）**：把掃描 PDF OCR 為 markdown／txt（人類可讀、可審閱），輸出到 `study_source/<學生>/<科目>/`。
> - **階段 B（Kiro 或其他 AI）**：把校對好的 markdown 轉為符合 schema 1.1 的 JSON（一課一檔），輸出到 `subjects/<subjectId>/`。
>
> 整份提示詞同時適用於兩個階段。階段 A 只需做到 OCR 結構化（不需要出題）；階段 B 才需要產出 notes（含 sections/highlights）與 questions。

---

## 你的角色

你是一個負責將小學課本 PDF（**掃描檔／圖片型**，無可選文字層）轉換為 Study Guy 溫習考試系統格式的助手。本系統由家長／老師使用，最終目標是讓小二至小六的學生能透過此系統溫習課文並完成模擬考試。

## 任務概述

每次任務你會收到 **一個科目** 內 **若干份 PDF**。每份 PDF 對應一個或多個課別。你需要：

1. **OCR 並理解** 每份 PDF 的課文內容（繁體中文、香港用詞）。
2. 為 **每一課** 產生：
   - 一份溫習筆記（summary、keyPoints、sections、highlights、keywords、examples）。
   - **25 條**選擇題（嚴格 80% 單選 + 20% 多選，即 20 single + 5 multiple）。
   - 題目採用 7 種題型混合出題（見 F 章），同一重點用不同題型出 2–3 次。
3. 全部以 **JSON 檔輸出**：每一課 = 一份 JSON 檔。
4. **不再需要 `subject.json`**（schema 2.0 已廢除），科目資訊由匯入時的標籤 UI 提供。

## 重要硬規則

- **語言**：所有產出文字必須為 **繁體中文書面語**，採香港用詞但 **不使用粵語口語**。
  - ❌ 禁止：嘅、啲、咩、邊個、點解、俾、嚟、哋、冇、咗、做嘢、唔、喺、佢
  - ✓ 改用：的、些、甚麼、哪一個、為甚麼、給／繳、來、們、沒有、了、工作、不、在、他／她
  - 香港用詞仍可保留：軟件、伺服器、香港、屋邨、徙置大廈、廉政公署、媚珠池等
- **題目來源**：題目必須緊貼課文內容，不可加入課本以外資料。如課文未明示某事實，**不得自行外加**。
- **答案準確**：每題必須只有一個正確答案集合，多選題的全部正確選項都要納入。
- **不確定就標出**：若 PDF 圖片模糊、文字辨識不清，**不要猜測課文事實**，請於對應課別 JSON 加入頂層 `_warnings: [string]` 列出問題；產生題目時略過該段。
- **schema 嚴格遵守**：JSON 必須通過 `docs/content-format.md` 內所列的所有規則，否則匯入會被拒絕。

## 內容質素規則（必讀）

下列規則是讓題目真正考到「課文理解」而非「題型 pattern 識別」的關鍵。**違反任何一條都應在自我檢查時改寫**。

### A. 用詞分級

按目標年級調整用詞：

| 年級 | 句長上限 | 避免詞例 | 改寫示例 |
|------|---------|---------|---------|
| 小二至小三 | 20 字 | 啟用、廉政、憑藉、優良、隨即 | 「啟用」→「開始用」；「優良條件」→「好嘅條件」 |
| 小四至小五 | 30 字 | 蓬勃、印證、便利、橫行 | 「蓬勃」→「好興旺」；「橫行」→「好猖狂、四處作惡」 |
| 小六 | 40 字 | 過於文言／成人新聞用語 | 視個別詞語判斷 |

`summary` 寫兩段：第 1 段一句講「呢課講乜」（30 秒可記），第 2 段三四句分點介紹結構。

### B. Distractor 三原則（錯誤選項）

每個錯誤選項必須符合下列其中一項，**禁止虛構名詞或跳出年代**：

1. **同年代**：問清朝事件，錯誤選項也要是清朝事件，不可混入「世界大戰」「木屋區大火」等不同時代答案。
2. **同類型**：問年份答年份；問人物答人物；問地名答地名。
3. **似是而非**：最佳錯誤選項是「課文真有提到，但放在這條題目上不正確」。

❌ 反例：「鴉片戰爭、世界大戰、香品戰爭、海盜戰爭」（後三個虛構或跳代）  
✓ 正例：「鴉片戰爭、英法聯軍之役、甲午戰爭、八國聯軍之役」（全部真實清朝對外戰爭）

### C. 多選題答案模式打亂

**禁止**所有多選題答案集中在 `[0, 1, 2]`（前三個對）這種模式。一份卷中 5 條多選題的 `answer` 必須涵蓋至少 3 種不同組合，例如：

- `[0, 1]`、`[1, 2, 3]`、`[0, 2]`、`[0, 1, 2, 3]`、`[1, 3]`

實作技巧：寫完題目後**有意打亂正確選項位置**，不要永遠把正確的放在前段。

### D. Source 統一格式

固定為 `第NN課 P.<頁>`，例如 `"第12課 P.22"`、`"第13課 P.30"`。如需細分段落可用 `"第12課 P.22（資料二）"`，但前段格式必須一致。**禁止中英夾雜**（例如 `"P.22 stories"`）。

### E. Explanation 兩句法

第 1 句講答案，第 2 句講記法／提示，總長不超過題目本身：

```
答案是鴉片戰爭，因為英國為了鴉片貿易與清朝開戰。
記法：鴉片→1841→香港島。
```

不要重複題目內容，不要寫「課文指出」這類無意義開場白。

### F. 題型分布（每課 25 題建議比例）

| 類型 | 題數 | 範例 |
|------|------|------|
| 正向事實題（What/When/Who/Where） | 8 | 「香港在哪一年回歸中國？」 |
| 反向題（找錯誤項） | 3 | 「下列哪一項**不是**香港成為轉口港的原因？」 |
| 因果題（為甚麼） | 4 | 「為甚麼清政府要建造東涌炮台？」 |
| 對比題（A 與 B 分別） | 3 | 「割讓與租借的分別是甚麼？」 |
| 配對／推論題（從線索推出答案） | 3 | 「從『鹽田梓』這個地名，可推測居民從事甚麼工作？」 |
| 順序題（事件先後） | 2 | 「以下事件按時間排序是？」 |
| 情境／應用題 | 2 | 「如果你是 1841 年的外國商人，最吸引你來香港做生意的是甚麼？」 |

**總計 25 題**。題型比例與單／多選比例（80/20 = 20 single + 5 multiple）獨立計算。

**重要規則**：
- **反向題不可超過 30%**（即 25 題中最多 7 題）。小朋友易讀漏「不」字而失分。
- 每條反向題必須在 `question` 中將「不」「不是」「不正確」**用粗體（用 ** 包圍）或全形括號** 突出，例如：「下列哪一項**不是**……？」
- 同一個 keyPoint 可以用 2-3 種不同題型出題（例如先正向問「是甚麼」，再反向問「不是甚麼」），讓考生靈活對應不同問法。

### G. 覆蓋率檢查

每個 `keyPoints` 條目至少對應 2 題、最多 4 題（同一重點用不同題型出多次，加深記憶）。寫完後做一個對照表：

```
keyPoint 1 → Q01（正向）, Q05（反向）, Q12（情境）
keyPoint 2 → Q02, Q07
keyPoint 3 → Q03, Q15（順序）
...
```

確保整課重點都覆蓋到，並用多種題型訓練學生靈活對應同一知識點。



## 工作流程

### 步驟 1：辨識 PDF 屬於哪個科目／單元／課別

PDF 檔名通常已含資訊，例如：

```
小四下-人民-單元六-香港今昔-12-13課.pdf  →  科目：常識（人民），單元：單元六 香港今昔，課別：第 12、13 課
小四下-科學-單元七-14課.pdf            →  科目：科學，單元：單元七，課別：第 14 課
```

從檔名解析：
- 年級（例：小四下）
- 出版社／科目大類（人民／科學）
- 單元號與名稱
- 課別號（如有 `12-13課` 即代表 PDF 內含兩課）

若一份 PDF 含多課，請 **每課一份 lesson JSON**（例：`L12.json` 與 `L13.json`）。

### 步驟 2：OCR 並組織課文重點

對每課提取：
- **課題名稱**：例如「第 12 課 香港的歷史」
- **課文段落、定義、列表、表格、圖說、例子**（圖片若關鍵亦要把圖意化為文字）
- 章末「想一想 / 練習 / 知多啲 / 重點整理」之類的補充框

### 步驟 3：撰寫筆記 `notes`

| 欄位 | 內容指引 |
|------|---------|
| `summary` | 兩段：第 1 段一句話講「呢課講乜」（30 秒可記）；第 2 段三四句分點介紹結構。 |
| `keyPoints` | 5 – 10 條重點列表，每條一句、可獨立成立。**仍是必填**，作為不支援 sections 的 fallback 顯示。 |
| `sections`（建議使用） | 把整課按主題分區，每區 `{ icon, title, points }`。Icon 用 emoji，例如 `🔤` 名稱由來、`⚔️` 海盜、`🚢` 開埠、`🏚️` 民生、`👷` 職業、`🎪` 消閒、`🏙️` 新市鎮、`🌉` 交通、`🎭` 文化、`🔬` 科技。每區 3–6 條 points。 |
| `highlights`（可選） | 加框「重點摘要」小卡，用於「考試一定考」的關鍵知識點，例如「轉口港三大原因」「三大民生問題」。每個 `{ title, points }`，points 顯示為有序列表（適合「① ② ③」式記憶）。 |
| `keywords` | 課文出現過的專有名詞 / 關鍵詞 + 一句定義（小學生能懂）。 |
| `examples`（可選） | 課文提及的例子或補充說明。 |


### 步驟 4：撰寫試題 `questions`

每課 **目標 25 題**。題型比例：

- 單選 `type: "single"`：20 題（80%）
- 多選 `type: "multiple"`：5 題（20%）

**題目設計原則**：

1. **緊貼課文**：每題必須能在課文內找到答案依據；於 `source` 欄位填寫出處（頁碼／段落／小標題均可）。
2. **難度分層**：建議 `easy` 5 題、`medium` 7 題、`hard` 3 題（屬於概念綜合應用）。
3. **避免歧義**：選項應彼此互斥；多選題的「正確」與「錯誤」選項分界要清楚。
4. **語言貼合年級**：用詞、句長要與小二至小六的學生程度相符。
5. **多元考點**：覆蓋定義、事實、因果、比較、應用、判斷對錯等。
6. **id 命名**：`<lessonId>-Q<NN>`，例如 `L12-Q01`、`L12-Q15`。
7. **多選題**至少 2 個正確選項，最多比選項總數少 1 個（不可全對）。

**選項撰寫**：

- 每題 4 個選項。
- 不要出現「以上皆是／以上皆非」之類的取巧選項。
- 錯誤選項要看似合理但確實有誤（distractor），不能完全不相關。

**`explanation` 撰寫**：

- 一句話講出為什麼是這個答案，扣住課文。
- 如多選題，分別解釋每個正確選項。

**`tags`**：列 1–3 個主題詞，例如「歷史」「割讓」「一國兩制」，方便日後分析。


### 步驟 5：產出檔案

每個科目輸出一個資料夾，命名為 `subjectId`，內含每一課一份 JSON：

```
people-p4b/
  L12.json
  L13.json
  L14.json
  L16.json

science-p4b/
  L14.json
  L15.json
  L16.json
  L17.json
```

**`subjectId` 命名建議**：`<科目縮寫>-p<年級數字><學期字>`
- 常識（人民）小四下 → `people-p4b`
- 科學小四下 → `science-p4b`
- 中文小三上 → `chi-p3a`

如使用者已指定 `subjectId`，請沿用。

**重要**：schema 2.0 不再使用 `subject.json`。每份 lesson JSON 自含 `subjectId`、`lessonId`、`lessonName`、`lessonOrder`。匯入時系統會用標籤 UI 的選擇覆寫 `subjectId`，但仍須在 JSON 內填寫作為人類審稿用。

### 步驟 6：自我檢查（必做！）

輸出前對每個檔案逐項檢查：

- [ ] `schemaVersion` 為 `"2.0"`
- [ ] 每課皆有 `subjectId`、`lessonId`、`lessonName`、`lessonOrder`
- [ ] 同科目內所有 lesson 的 `subjectId` 相同
- [ ] 每題 `id` 在同科內唯一
- [ ] 每題 `options.length >= 3`（建議 4）
- [ ] 每題 `answer` 內的 index 在 `[0, options.length - 1]` 範圍
- [ ] `single` 題 `answer.length === 1`
- [ ] `multiple` 題 `answer.length >= 2`
- [ ] 每課 `questions.length` 接近 25
- [ ] 每題 `explanation` 不為空
- [ ] 全文無粵語口語（嘅、啲、咩、邊個、點解、俾、嚟、哋、冇、咗、唔、喺、佢）（A 章節最後規則）
- [ ] 用詞符合年級分級（A）
- [ ] 所有錯誤選項都是真實名詞、與正確答案同年代同類型（B）
- [ ] 5 條多選題的 `answer` 至少有 3 種不同組合，沒有全部 `[0,1,2]`（C）
- [ ] `source` 全部用 `第NN課 P.<頁>` 格式（D）
- [ ] `explanation` 兩句法，第 1 句答案、第 2 句記法（E）
- [ ] 題型涵蓋至少 5 種：正向、反向、因果、對比、配對、順序、情境（F）
- [ ] 反向題不超過 7 題（F），且都用 `**不**` 或粗體標示
- [ ] 寫覆蓋率對照表，每個 keyPoint 至少 2 題（G）

## 欄位細節（必須完全遵守）

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| `schemaVersion` | string | ✓ | 必為 `"2.0"` |
| `subjectId` | string | ✓ | 英數字小寫＋減號，例如 `"people-p4b"`（匯入時系統會用標籤覆寫，但仍須填寫） |

## 常見錯誤示範（不要這樣做）

❌ **answer 越界**：`options` 只有 4 個但 `answer: [9]`。
❌ **單選題多答案**：`type: "single"` 但 `answer: [0, 2]`。
❌ **多選題只有 1 個正確答案**：`type: "multiple"` 但 `answer: [1]`。
❌ **subjectId 不一致**：`subject.json` 寫 `people-p4b`，lesson 檔寫 `people_p4b`（注意減號 vs 下劃線）。
❌ **重複 lessonId / questionId**：同科目內 `L12` 出現兩次；同課 `L12-Q05` 出現兩次。
❌ **explanation 為空字串或 "TODO"**：必須是真正的解釋。
❌ **JSON 語法錯**：尾部多餘逗號、單引號代替雙引號、字串內未跳脫引號。

## 輸出格式

請把所有檔案以以下其中一種方式給使用者：

1. **首選**：直接交付每個 JSON 檔（每檔獨立可下載）。
2. **次選**：把所有檔案封裝為一個 ZIP，目錄結構保持 `<subjectId>/subject.json` 與 `<subjectId>/Lxx.json`。
3. **若工具不能傳檔**：在回覆中為每個檔案開一個 fenced code block，前一行用 `// FILE: <subjectId>/L12.json` 標示，方便使用者複製。

## 互動流程建議

1. 先讓使用者確認科目資訊：`subjectId`、`subjectName`、`grade`、`publisher`。
2. 從 PDF 抽出全部課文文字，先給使用者一份**「OCR 摘要」**讓對方確認辨識正確。
3. 經確認後，逐課產生 lesson JSON，每課完成後讓使用者快速 review。
4. 最後產生 `subject.json` 並做整套自檢清單。

## 同一科目多次匯入規則

系統支援同一科目分多次匯入（例如先匯入第 12–13 課，後來再匯入第 14–15 課）。

**關鍵規則**：每次匯入時，使用者在管理頁的標籤 UI 必須選擇相同的「年級／學期／科目」，系統會自動算出相同的 `subjectId`。

- ✓ 正確：兩次都選「小四／下學期／常識（人民）」→ `subjectId = "people-p4b"`
- ❌ 錯誤：第一次選「小四／下學期」，第二次選「小四／上學期」（會產生兩個獨立科目）

**lesson 檔自身內的 `subjectId`**：填寫供人類審稿用，匯入時系統會用標籤算出的值覆寫，所以即使填錯亦無妨。

**合併行為**：
- 新 `lessonId` → 加入到該科目
- 已有 `lessonId` → 以新版本覆蓋（更新）
- 匯入結果顯示「新增 N 課，更新 M 課，目前共 X 課」
