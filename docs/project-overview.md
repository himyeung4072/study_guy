# Study Guy 專案目的

## 專案概述

建立一個本地運行、可擴充的 HTML 溫習及模擬考試系統，協助學生（小二至小六）就不同科目進行溫習與考試，並透過累積考試記錄分析學生弱項，於下一次考試針對性加強練習。

系統採用 **資料與程式分離** 設計：所有課文重點及試題以特定 JSON 格式儲存（**一份 JSON 檔 = 一課**，schema 2.0 已廢除 `subject.json` 與單元結構），程式只負責載入、顯示、出題、評分及分析。將來新增科目／年級，只需新增 lesson JSON 檔即可。

## 使用者

- 主要使用者：小二至小六學生，可有多位（家中兄弟姊妹共用同一部電腦）
- 內容編輯者：家長／老師，使用其他工具（PDF 文字抽取、AI 整理等）將課文輸出成本系統定義的 JSON 格式

## 核心功能

### 1. 科目／內容管理
- 系統支援 **多科目**（常識、科學，將來可加中、英、數等任何科目）。
- 內容以 **一課一個 JSON 檔** 形式儲存（schema 2.0），每課同時包含 **subjectId、溫習筆記重點、試題**。
- 程式啟動時讀取已匯入的科目；亦可隨時透過介面 **匯入** 新課別／更新現有課別。
- **同一科目多次匯入自動合併**：相同 `subjectId` 的課別，新課別會加入、已有課別會更新，不會出現重複科目。
- **標籤式匯入 UI**：管理頁提供年級／學期／科目／章節標籤選擇器，自動算出 `subjectId`，無需手動填寫。標籤可自由新增或刪除（儲存於 `localStorage`）。
- **不再需要 `subject.json`**：科目資訊由匯入時的標籤 UI 提供。

### 2. 溫習筆記
- 系統根據匯入內容，按 **科目 → 課別** 顯示溫習筆記重點（schema 2.0 已扁平化，不再分單元）。
- 筆記為精簡重點（摘要、key points、sections、highlights、關鍵詞、例子），方便學生快速溫習。
- 學生可選擇某一課專心溫習。

### 3. 模擬考試
- **單科考試**：每次考試只考一個科目。
- **題數**：固定 **25 題**。
- **題型比例**：約 **80% 單選 + 20% 多選**（即 25 題中 5 題為多選）。
- **抽題策略**：
  - 預設於該科所有課別之間 **平均抽取**。
  - 若該考生有「弱項課別」（見第 5 點），抽題比例調整為 **弱項 50% + 平均 50%**。
- **考試過程**：不顯示答案，不限時，提交後一次過評分。每次抽題時選項順序隨機洗牌（`answer` index 同步調整），避免學生背答案位置。
- **離開考試**：考試頁有「離開考試」按鈕，確認後清除進度返回主頁。
- **考試結束**：顯示分數、每題對錯、正確答案、解釋及課文出處，並按課別列出表現。

### 4. 考生管理
- 多考生支援：可建立／選擇已有考生帳號。
- 建立考生時必須填寫年級（小一至小六）。
- 主頁科目選擇只顯示與考生年級相符的科目（例：考生「小四」可看「小四上」「小四下」）。
- 每個考生獨立儲存歷史記錄、弱項分析。

### 5. 錯題分析與針對性出題
- 每次考試記錄分數、答題詳情、用時、日期。
- 分析每課的正確率：**正確率低於 60% 視為弱項**。
- 下一次考試自動將弱項課別出題比例提升至 50%。
- 累積分析提供：
  - 整體分數趨勢
  - 各課強弱熱圖
  - 持續錯誤的試題列表

### 6. 本地儲存
- 所有資料（已匯入的科目、考生、考試記錄）儲存於瀏覽器 `localStorage`。
- 提供匯出／匯入 JSON 作備份。

## 介面與風格

- 語言：**繁體中文（香港用詞）**。
- 適合 **小二至小六** 學生使用：大字體、清晰排版、有少量圖示與顏色，不過度幼稚化以照顧較高年級。
- 介面分區：
  - 主頁：選擇考生（含年級）、選擇科目（按年級過濾）
  - 溫習頁：列出該科所有課別（按 lessonOrder 排序）→ 點某課顯示筆記
  - 考試頁：答題介面（含「離開考試」按鈕）
  - 結果頁：成績及錯題分析
  - 歷史頁：累積記錄與弱項分析
  - 管理頁：標籤式匯入課別、新增／刪除考生、備份／還原（**需輸入密碼進入**）

## 內容生成工作流程

兩個階段：

### 階段 A：PDF（掃描檔）→ Markdown（人類審閱）

由使用者（家長／老師）在外部使用任何具視覺能力的 AI Agent（ChatGPT／Claude／Gemini 等），以 `docs/agent-prompt-pdf-to-json.md` 內 **第一部分（OCR + 結構化）** 為提示詞，將 `study_source/<學生>/<科目>/*.pdf` OCR 並整理為 markdown／txt 純文字檔，產出位置：

```
study_source/<學生>/<科目>/<課文主題>_<單元>_課程內容.md
```

例如：
```
study_source/hayson/people/香港今昔_單元六_課程內容.md
```

此 markdown 應包含：每課的標題、分節、重點段落、關鍵詞、例子、年份事件等，**保留人類可讀的編輯餘地**（先審稿、修錯字、補不清楚之處）。

### 階段 B：Markdown → JSON（Kiro 自動轉換）

使用者把整理好的 markdown 提交給 Kiro，Kiro 依照 `docs/agent-prompt-pdf-to-json.md` 內 **第二部分（質素規則）**，把 markdown 轉為符合 schema 2.0 的 lesson JSON：

```
subjects/<subjectId>/
  Lxx.json
  Lxx.json
```

注意：schema 2.0 不需要 `subject.json`。每份 lesson JSON 自含 `subjectId`、`lessonId`、`lessonOrder`，匯入時由標籤 UI 提供 subjectName 與 grade。

最後再經由「管理 → 匯入課別」上載到本地 `localStorage`。

### 內容格式

詳見 `docs/content-format.md`（schema 2.0：一份 JSON = 一課，無 subject.json，notes 含 sections/highlights）。

範本檔案位於 `tools/lesson-template/`。

## 技術方向

- 純前端：HTML + CSS + 原生 JavaScript（不依賴框架，方便雙擊 `index.html` 即用）
- 內容：JSON（一課一檔，schema 2.0），匯入後存入 `localStorage`
- 儲存：`localStorage`（考生資料、考試記錄、弱項分析、已匯入科目、標籤設定）
- 內容載入方式：透過介面 **匯入** lesson JSON 檔（拖放或檔案選擇器）

## 跨瀏覽器與裝置支援

- **桌面**：Chrome / Edge / Firefox / Safari（皆支援，主要使用情境）
- **iPad / iPhone（Safari）**：完整支援，已加入下列 iOS 優化：
  - 自製 Modal API（取代 `prompt`/`confirm`，解決 iOS PWA 模式擋對話框問題）
  - PWA manifest：iPad / iPhone 可從 Safari 點「分享 → 加入主畫面」變全螢幕 App
  - 匯出備份提供「下載 + 複製到剪貼簿」雙路徑（iOS Safari 下載不穩定時可改用複製）
  - 觸控優化：所有按鈕 ≥ 44×44px，符合 iOS HIG 建議
  - 響應式：iPad 直向 tile-grid 自動改單欄
  - safe-area-inset：iPhone 瀏海／圓角自動避讓
  - input font-size: 16px，防 iOS 點 input 時自動縮放畫面
