# Context7 MCP 編碼輔助規則

## 規則說明

所有新增或修改的程式碼，必須先使用 Context7 MCP 來協助查詢相關文件和最佳實踐。

## 執行步驟

1. **查詢前置作業**：在編寫或修改任何程式碼之前，先使用 `mcp_Context7_resolve_library_id` 工具來解析相關的函式庫 ID
2. **查詢文件**：使用 `mcp_Context7_query_docs` 工具查詢該函式庫的最新文件、範例和最佳實踐
3. **應用知識**：根據查詢結果來編寫或修改程式碼

## 適用範圍

- 新增功能程式碼
- 修改現有程式碼
- 引入新的函式庫或框架
- 更新依賴版本

## 範例流程

```
1. 識別需要使用的技術/函式庫（例如：React, TypeScript, Vitest）
2. 呼叫 resolve_library_id 取得 Context7 相容的 library ID
3. 呼叫 query_docs 查詢相關文件
4. 根據文件內容進行編碼
```

## 注意事項

- 每個問題最多呼叫 Context7 工具 3 次
- 優先使用官方文件而非部落格文章
- 確保引用來源的正確性
